# 🌐 3D 시각화 플랫폼 확장 계획 (Expansion Beyond Physical Objects)

> **상태**: 초안 (v0.1)  
> **핵심 가치**: 실물 기물(의류/가구)을 넘어 추상적 개념과 소프트웨어 구조를 시각화하여 서비스의 확장성과 ROI를 극대화함.

---

## 1. 확장 배경 및 필요성

기존의 '가상 피팅(Virtual Try-on)'은 의류라는 특정 기물에 국한되어 시장의 변동성과 기술적 특수성에 매몰될 위험이 있습니다. 이를 **"추상적 데이터의 물질화(Materialization of Abstract Data)"**로 확장함으로써 다음과 같은 기대효과를 얻을 수 있습니다:

1.  **시장 확장성**: 패션/커머스를 넘어 에듀테크, IT 인프라 모니터링, 데이터 분석 시장으로 진입 가능.
2.  **리소스 효율**: 구축된 3D 렌더링 엔진(Three.js)과 데이터 처리 파이프라인(Python/Java)을 다양한 도메인에 재사용.
3.  **차별화된 가치**: 눈에 보이지 않는 소프트웨어 로직이나 복잡한 JSON 구조를 입체적으로 시각화하여 이해도를 높임.

---

## 2. 시각화 대상의 확장 (Use Cases)

### A. 소프트웨어 & 데이터 구조 (Software & Data)
*   **JSON/XML 계층 구조**: 복잡하게 얽힌 JSON 데이터를 3D 노드 트리 형태로 시각화하여 데이터 간의 관계를 탐색.
*   **시스템 아키텍처 맵**: 마이크로서비스 간의 통신, API 호출 흐름을 3D 공간의 '빛의 흐름'으로 표현.
*   **DB 스키마 & 관계**: 테이블 간의 관계(ERD)를 3D 공간에 배치하여 복잡한 조인 구조를 한눈에 파악.

### B. 이론적/추상적 개념 (Theoretical Concepts)
*   **알고리즘 시나리오**: 정렬, 탐색, 재귀 등의 알고리즘이 실행되는 과정을 3D 애니메이션으로 시각화.
*   **AI 모델 내적 구조**: 신경망의 가중치 변화나 벡터 공간상의 데이터 분포를 입체적으로 표현.
*   **보안 인증 프로세스**: JWT 토큰의 발행, 검증, 탈취 방지 로직을 가상의 성벽과 열쇠 시나리오로 시각화.

---

## 3. 기술적 구현 방향

### Python Backend (Data Processor)
*   **역할 변경**: 'AI 가상 피팅 서버' → **'3D 시나리오 생성기(Scenario Generator)'**.
*   **기능**: 추상적 데이터(JSON, Log, Code)를 입력받아 Three.js에서 읽을 수 있는 좌표(JSON/GLB)와 속성값으로 변환.
*   **추가 모듈**: 데이터 파서(Parser), 좌표 매핑 엔진, 시나리오 스크립터.

### Java Backend (Core Management)
*   **역할**: 시각화 템플릿 및 권한 관리.
*   **기능**: 사용자별 시각화 프로젝트 저장, 결제 모델(구독형 시각화 도구) 연동.

### Frontend (General 3D Viewer)
*   **역할**: 인터랙티브 3D 대시보드.
*   **기능**: Three.js 기반의 범용 3D 엔진 구축. 기물(옷)뿐만 아니라 노드, 링크, 파티클 등을 자유롭게 렌더링.

---

## 4. 로드맵 반영 사항

1.  **PoC 단계**: JSON 데이터를 입력받아 3D 공간에 노드로 배치하는 간단한 실험 진행.
2.  **MVP 단계**: 의류 시각화와 병행하여, 시스템의 '로그 데이터'를 3D로 보여주는 대시보드 기능 추가.
3.  **고도화 단계**: 사용자가 직접 시각화 형식을 커스터마이징할 수 있는 '시각화 빌더' 개발.
