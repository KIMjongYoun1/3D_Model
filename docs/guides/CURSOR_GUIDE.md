## ⚠️ 추가 과금 발생 상황

### 과금이 발생하지 않는 경우 (현재 Pro 기준)

- ✅ 파일 읽기/쓰기
- ✅ 터미널 명령 실행
- ✅ 웹 검색
- ✅ Slow Request 사용 (무제한)

### 과금이 발생할 수 있는 경우

| 상황 | 설명 | 대응 |
|------|------|------|
| Fast Request 500회 초과 | 추가 요청 시 과금 가능 | Slow로 전환 |
| Business 플랜 필요 기능 | 팀 협업, 관리 기능 | 혼자 개발 시 불필요 |

### 💡 비용 절약 팁

1. **간단한 질문은 Slow Request로**
   - 문서 작성, 간단한 수정
   - 대기 시간 있지만 무제한

2. **핵심 작업에만 Fast Request**
   - 복잡한 기능 구현
   - 디버깅, 에러 해결
   - 아키텍처 설계

3. **컨텍스트 효율적 사용**
   - 한 번에 여러 파일 수정 요청
   - 명확한 요구사항 전달로 재요청 최소화
